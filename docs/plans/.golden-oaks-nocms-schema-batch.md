# P9 PRE-IDENTIFIED FINDING — responsive breakpoint convention (fix in P9 Task 3)

**Tailwind v4 gotcha (confirmed empirically by the P4 content-blocks impl):** `max-[768px]:` and
`max-[480px]:` compile to `@media (width < 768px)` / `(width < 480px)` — **EXCLUSIVE**. So a layout
flip keyed on `max-[768px]:` does NOT apply at exactly 768; the desktop layout shows. But the mockup
uses `@media (max-width: 768px)` — **INCLUSIVE** (applies AT 768). The P9 audit screenshots at EXACTLY
{1440, 1024, 768, 480}, so any block flipping on `max-[Npx]:` (or on bare `md:`/`lg:` min-width, which
is desktop-at-the-boundary) renders the WRONG layout step at the exact audit width vs the mockup.

**Correct conventions already used by most blocks (leave as-is):**
- `min-[769px]:` / `min-[481px]:` / `min-[1025px]:` — mobile-first base + desktop override at ≥N+1.
  Base (mobile) correctly applies at ≤768/≤480/≤1024 inclusive. **CORRECT.** (~127 uses.)
- `[@media(max-width:768px)]:` / `[@media(max-width:480px)]:` — content-blocks' inclusive arbitrary
  form. **CORRECT.** (~14 uses.)

**At-risk (P9 must audit + reconcile each at the exact boundary):**
- `max-[768px]:` (~25), `max-[480px]:` (~42), `max-[1024px]:`/`max-[640px]:` (1 each) — exclusive;
  off-by-one at the exact boundary. Convert to the inclusive `[@media(max-width:Npx)]:` form OR
  re-express as a `min-[N+1]:` complement.
- Bare `md:`/`lg:` grid/flex flips across ~20 block + layout files — `md:`(≥768)/`lg:`(≥1024) show the
  DESKTOP step AT 768/1024; if the mockup's flip is inclusive at that boundary, off-by-one. Many are
  fine (the mockup boundary may be 960/640/600, not 768/1024) — judge per-block against the mockup.
  The Header hamburger swap (`lg:hidden` / `hidden lg:flex`) is one to check at exactly 1024.

**P9 Task 3 approach:** run the matrix audit at {1440,1024,768,480}; for any cell where the template
shows a different layout STEP than the mockup at the exact width, fix the owning block's breakpoint
form (prefer the inclusive `[@media(max-width:Npx)]:` or `min-[N+1]:` complement), re-run that page.
This is a known, bounded sweep — not a per-page surprise. Token values are unaffected; this is purely
the media-query boundary.
